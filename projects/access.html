Databases

During my time at Linn County, I began working on databases using Microsoft access. I adopted a couple databases from a former co-worker, then began creating my own as needs arose.

These included a SQL database, VBA programming, and front-end design.


Click on images below to see full size.

PEO Database

The PEO (Precinct Election Official) database was used to assign workers to locations for election day. It also allowed us to sign workers up to training and calculate payments. It was one of the databases I inherited. However, a lot of the content was reworked and several features were added.

This was the bulk of my duties when I was the Election Office Coordinator, so I spent a lot of time with it. I also added several things to manage and generate documents that were used for election day.

Problem Log

We had to keep track of incoming calls and other issues on election day. We had several people on the phone, so we needed a ticketing system. This database replaced Google Forms as our go-to.

Some features:

  • Searching a location based on partial name or precinct code.
  • Automatically pulling info for the chairperson at that location, and allowing you to copy that info into the ticket, if they are the one calling.
  • A built-in micro website to help with troubleshooting, which will automatically pull up the applicable page when you select "Type of Problem".
  • Links to pdf guides
  • A history of tickets you submitted, that you can click on for details / updates.
  • An admin page so tickets could be escalated and managed by senior members of the team.
  • Automatic timestamps and usernames applied to those performing actions on the tickets.
  • A built-in chat.

Jira Task List

We used Jira to manage our task lists for each election. Unfortunately, at the time, there was no way to copy tasks from one project to another, so I made a database to do just that.

This also allowed me to set up tasks for specific election types, I could filter out tasks and change deadlines depending on what type of election it was.

Write-In Database

After each election with a write-in vote, our office had to tally each write-in vote, which involved typing them out manually. We had several temporary workers do this. This database made that process a little less painful.

It also handled variations of names (ex. jon, john, jonathan), so each would count for the same write-in vote. A report could then be generated for the public to view.

Depot Drop-Off Database

At the end of election night, all the election locations have a chairperson who brings equipment back to us. In order to ensure we have everything, we barcoded everything and I made this database to quickly enter the barcodes and generate a report of what was scanned.

Polling Place Database

The Polling Place database was the other database I inherited with the PEO Database. This one didn't receive as much love.

Items I worked on that I don't have screenshots for:

  • A repository for ADA survey documents.
  • details on each polling location

Contacts Database

This database was created to maintain a list of temporary workers we used. It could also be expanded to any contacts as needed.

Inventory Database

One of my tasks as Elections Office Coordinator was keeping track of office and election supplies. The inventory database allowed me to keep track of where things were and what we had. This helped verify we had enough materials for upcoming elections.

Turnout Calculator

We needed to estimate turnout for each election. My solution was to make a database, which takes historical data and updates for current registration counts.

This one was just for me, so it looks kind of basic, but you could select which prior elections to base it on, add some padding, and determine how many check-in workers you would need.

Yellow Bag Database

The yellow bag contained materials necessary on election night. This was created to help us keep track of which bags we received, verify their seal numbers (barcodes) matched up, and verify vote totals.
