Jira and Confluence

Jira and Confluence are cloud-based software tools by Atlassian.

Jira is used for project management. You can assign tasks to workers and set deadlines.

Confluence is basically a wiki. It is a reference tool on how to perform tasks and can host needed documents.

I had used Confluence extensively at ATT and ended up bringing it to Linn County, where I ended up implementing Jira as well.


Jira

For each election, I would create a Jira project of tasks for the whole office (previously a list in Excel by the deputy).

Most of these tasks were similar for each election, depending on different situations, like: If early voting outside the office would occur. Many tasks had different deadlines based on the type of election. Because of these things, I ended up making an Access database to generate tasks and deadlines for elections.


No fancy pictures here. It looked like any other Jira instance, I'm sure.

Confluence

Confluence was the spot for sharing knowledge. Just about everything I had worked on was in Confluence, so myself and others could quickly search for the info.
