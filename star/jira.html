Software Improvement:
Jira Task List

Situation

Initially, our deputy used an Excel spreadsheet to manage election tasks and we had a regular meeting to go through each item and say what was and wasn't completed.

In comes Jira (Core) to allow us to manage these tasks better, but Jira didn't have a good way to duplicate the tasks from election to election. Also, not all tasks were the same or had the same deadlines for each election. Lastly, Jira didnâ€™t have a decent export / import function at the time.

Task

We needed a way to duplicate elections, at least. Better yet, we needed to import elections based on the type of election, with dynamic deadlines based on the days prior to election day.

Action

I created an access database to add tasks, with options to tag them in various ways:

  • Triggers: When the task can be started.
  • Dependencies: What type of election or what needed to take place in an election for the task to apply, like early voting.
  • Links: Connecting related issues and potentially triggering alerts when an issue is completed.

Tasks had deadlines related to election day (-120 days, for example) which updated based on the date I added for the election.

I could filter out dependencies to quickly filter out several tasks.

Then I could export the file as a CSV and import it into Jira.

Result

I was able to create a Jira project for an election with hundreds of tasks and accurate deadlines within an hour. All tasks were assigned to the correct worker, had applicable notes, and were linked to other tasks as necessary.

This all served to improve collaboration between the election team as we were in busy election times.
